# Onoma

A web app that solves the beatmaker's naming block: after finishing a beat, producers get stuck choosing a name or settle for something bad. Onoma generates catchy names anchored to the beat's traits so naming takes seconds instead of stalling the workflow.

## Language

**Naming Block**:
The moment a beatmaker finishes a beat and stalls on choosing a name — the core problem Onoma solves.
_Avoid_: Forgetting, recall problem

**Beat Name**:
An AI-generated name for a beat that is catchy/evocative yet derived from the beat's traits, so it fits the beat rather than being random.
_Avoid_: Random name, title

**Batch**:
The result of one generation: ~10 Beat Names in deliberately mixed naming styles, produced from a single Beat Profile. The user picks from the batch or regenerates.
_Avoid_: Results, suggestions, list

**Beat Profile**:
The description a beatmaker fills in to anchor name generation: required free-form Tags, plus optional BPM, reference artist, and vibe notes. The beat may not exist yet — profiles are also filled speculatively for inspiration. Audio upload may later populate the profile automatically.
_Avoid_: Input, description, metadata

**Tag**:
A free-entry chip describing the beat in the beatmaker's own words — genre, mood, sound, or any blend ("dark trap", "eerie bells", "hyperpop"). Tags are not a fixed taxonomy; suggestion chips exist only as shortcuts.
_Avoid_: Genre, mood, category

**Name-First Inspiration**:
The secondary use case: generating a Beat Name before the beat exists, to spark a creative direction.
